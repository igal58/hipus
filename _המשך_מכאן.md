## API של השרת (כולם מאותו origin)
> **פרמטרים משותפים חדשים:** `cur` (ils/usd/eur — ברירת מחדל ils) נתמך ב-offers/price/deals/calendar/flexmonths.
> `stops` (0–9, מקס׳ עצירות) נתמך ב-offers/price. השרת מסנן `transfers<=stops`.
- `GET /health` → `{ok, tokenConfigured, cloud}`.
- `GET /price?origin=&destination=&depart=YYYY-MM-DD[&return=YYYY-MM-DD]` → הטיסה **הזולה ביותר** (טיסה אחת). אם `return` ריק → חיפוש **דו-כיווני גמיש** (יציאה קבועה, חזרה פתוחה). מחזיר גם `departISO/returnISO/flex/departTime/returnTime`.
- `GET /offers?origin=&destination=&depart=[&return=]` → **עד 6 טיסות אמיתיות שונות** (מחיר/חברה/שעות/עצירות מגוונים), כל אחת עם קישור רכישה (marker). זה מה שהדף משתמש בו לתוצאות.
- `GET /deals?origin=TLV` → הזול מתל אביב לכל העולם (מבצעים חמים).
- `GET /calendar?origin=&destination=&month=YYYY-MM` → **מחיר זול לכל יום בחודש** (Travelpayouts `grouped_prices`, `group_by=departure_at`). מחזיר `{days:[{date,price,airline,transfers,link}], cheapest}`.
- `POST /alerts` (JSON: `email,origin,destination,depart,ret?,target`) → יוצר התראת מחיר. `GET /alerts?email=` → רשימת ההתראות של המייל. `GET /alerts/delete?id=&email=` → מחיקה.
- `GET /check-alerts?key=CRON_KEY` → בודק את כל ההתראות מול מחיר חי ושולח מייל למי שירד ליעד (one-shot). מופעל ע"י GitHub Action / cron חיצוני. מוגן ב-`CRON_KEY` אם מוגדר.
- `GET /flexmonths?origin=&destination=&from=YYYY-MM-DD&months=4` → **הזול ביותר בכל אחד מ-N החודשים הקרובים** (חיפוש גמיש). מחזיר `{months:[{month,date,price,airline,link}], cheapest}`.

## הקמת התראות מחיר (כדי שיעבדו בענן — עדיין לא הוקם)
ה-MVP בנוי ועובד מקומית ב-dry-run. כדי שישלח מיילים אמיתיים בענן צריך 3 דברים:
1. **מפתח Resend** (חינמי, resend.com): צור API key, והגדר ב-Render env: `RESEND_API_KEY=...`. אופציונלי `ALERT_FROM="Hipus <דומיין-מאומת>"` (ברירת מחדל onboarding@resend.dev עובד לבדיקות).
2. **CRON_KEY** (סוד כלשהו) ב-Render env — מגן על `/check-alerts`.
3. **תזמון אוטומטי** — כבר מובנה ע"י `.github/workflows/price-alerts.yml` (רץ כל 3 שעות, מעיר את Render + מפעיל בדיקה). רק צריך: ב-GitHub → Settings → Secrets → Actions → להוסיף `CRON_KEY` (זהה ל-env ב-Render). חלופה: cron-job.org ל-`/check-alerts?key=<CRON_KEY>`. בנוסף יש מתזמן פנימי בשרת (`CHECK_HOURS`, ברירת מחדל 6ש) שרץ כשהשרת ער.
⚠️ **אחסון**: כרגע `alerts.json` (gitignored, מכיל מיילים). ב-Render חינמי הקובץ **ארעי** — נמחק בכל deploy/קור-סטארט. לפרודקשן אמיתי: לעבור ל-Upstash Redis (REST, חינמי) או DB. השרת מוכן להחלפת `loadAlerts/saveAlerts` בלבד.

## מה כבר עובד (הושלם)
1. **קישורים חיים** ל-Google/Skyscanner/Kiwi/Kayak/Momondo + **El Al** — בשורת "השוואה וחיפוש באתרים אחרים" (כפתורים **בלי מחיר**).
2. **~233 ערים** ב-3 שפות; חיפוש חוצה-שפות + IATA.
3. **תוצאות = כמה טיסות אמיתיות שונות** (`/offers`, עד 6) — מחירים/שעות/חברות מגוונים, לא אותו מחיר לכולם. כל כרטיס עם כפתור "רכישה ב-Aviasales" (עמלה). הזול מסומן תווית **"משתלם"** (לא "הזול ביותר", כי מקור אחד).
4. **שעות יציאה/חזרה** + **"ללא כבודה רשומה"** על כל כרטיס ובסיכום (אייקוני Font Awesome, בלי אמוג'י).
5. **ימי שהייה = 0 → דו-כיווני גמיש**: לא ממלא תאריך חזרה, נשאר 0; מחפש את הזול מהתאריך והלאה; מסומן "תאריכים גמישים"; במצב זה **לא מציגים זמן חזרה**.
6. **בורר נוסעים** (מבוגרים/ילדים/תינוקות) + **בורר כבודה** — באותה שורה עם כפתור החיפוש.
7. **מבצעים חמים** (`/deals`).
8. **תיקוני IATA**: KIV→**RMO** (מולדובה), ULN→**UBN** (אולן באטור), FRU→**BSZ** (בישקק/קירגיזסטן — Travelpayouts לא מכיר FRU), PNH→**KTI** (פנום פן/קמבודיה — נמל Techo החדש; PNH הישן "not flightable"). **ביקורת אמיתית** של כל 233 הקודים בוצעה ע"י קריאה בפועל ל-`prices_for_dates` (לא רק autocomplete!) — רק FRU ו-PNH היו שבורים, שניהם תוקנו. כל ה-231 הנותרים תקינים.
9. **שמות חברות תעופה**: מפה `AIRLINE_NAMES` (55 חברות, 3 שפות) ממירה קוד IATA (LY/EY/W6…) לשם מלא ("אל על"/"אתיחאד"). מוצג בכרטיס ובסיכום. קוד לא מוכר → מוצג כפי שהוא (בלי להמציא).
10. **"לאן שבא לי" לפי תקציב**: שדה תקציב (₪) ליד כפתור המבצעים → מסנן את `/deals` לפי הסכום, כותרת "לאן שבא לי — עד X ₪", הודעה ייעודית כשאין יעדים. ריק = כל המבצעים.
11. **לוח מחירים / החודש הזול**: כפתור "📅 החודש הזול" → רשת ימים עם מחיר זול לכל יום (לפי היעד הנבחר + חודש תאריך היציאה), הזול מודגש "הכי זול", לחיצה על יום ממלאת תאריך ומריצה חיפוש. משתמש ב-`/calendar`.
12. **קישור אל על תלוי-שפה**: `heb/israel` בעברית, `eng/israel` באנגלית/רוסית (rus/israel מחזיר 492). אין deep-link ציבורי יציב עם מסלול/תאריכים — לא ממציאים פרמטרים. אם המשתמש יספק URL אמיתי מחיפוש באתר אל על → אפשר לבנות deep-link מדויק.
13. **התראות מחיר (MVP, מייל)**: טופס "🔔 התראת מחיר" אחרי כל חיפוש (מייל + מחיר יעד למסלול הנוכחי). שרת שומר ב-`alerts.json`, שולח מייל דרך **Resend**. `/check-alerts` בודק מול מחיר חי ושולח כשירד ליעד (one-shot). בלי `RESEND_API_KEY` → מצב **dry-run** (לא שולח, רק מדווח — לבדיקות). מייל אחרון נשמר ב-localStorage. ראה "הקמת התראות" למטה.
14. **ימי שהייה=0 — הסבר + חלון רב-חודשי**: הערה תחת השדה מסבירה "0 = חיפוש גמיש מתאריך היציאה והלאה" (מודגשת כשהערך 0). בנוסף, חיפוש גמיש מציג **חלונית "כמה חודשים קדימה"** (`/flexmonths`) — הזול בכל אחד מ-4 החודשים הקרובים, הזול הכולל מסומן, לחיצה על חודש מחפשת לתאריך הזול שלו.
15. **התראות אוטומטיות בלי מסך**: (א) מתזמן פנימי בשרת (`setInterval`, ברירת מחדל כל 6ש, `CHECK_HOURS`) שרץ כשהשרת ער + self-ping (`RENDER_EXTERNAL_URL`) שמצמצם שינה. (ב) **GitHub Action** `.github/workflows/price-alerts.yml` שרץ כל 3 שעות, מעיר את Render וקורא ל-`/check-alerts` — עובד גם בלי מסך פתוח. צריך להגדיר secret `CRON_KEY` ב-GitHub (זהה ל-env ב-Render).
16. **פייסליפט — נאבבר + מותג**: נאבבר sticky עם מותג **FlyFinder** (סמל מטוס SVG + שם + כתובית), פעולות ניווט (חיפוש/מפה/מבצעים/החודש הזול/התראה — גוללות/מפעילות פונקציות קיימות), בורר שפה, וכפתור התאמה אישית. במובייל: **סרגל ניווט תחתון**. הכל מתורגם 3 שפות, RTL. ה-`<header>` והכפתורים המרחפים הישנים (נקודות צבע, toggle בהיר/כהה) **הוסרו**.
17. **מגירת "התאמה אישית"** (`#custDrawer`, toggle מהנאבבר): צבע מותג (6: blue/teal/purple/rose/orange/neutral), רקע (light/dim/dark), גודל טקסט (small/medium/large). חל דרך CSS variables (`applyCustomize()`), נשמר ב-localStorage, נסגר ב-X/רקע/Esc/כפתור.
18. **בורר מטבע ₪/$/€** (`currentCur`): בשורת כותרת התוצאות. עובר כ-`cur` לכל ה-API; הסמל (`curSym()`) מוצג בכל מקום. החלפה מרעננת מיד.
19. **מקס׳ עצירות (0–9)** + **בורר מחלקה** (Economy/Comfort/Business/First) + **נוסעים** — שלושתם בשורת פילטרים אחת (`.filters-row`). שינוי כל אחד → חישוב מיידי.
20. **מחיר שמתחשב בפרמטרים**: המחיר המוצג = מחיר-לנוסע × `paxCount()` × `classMult()`. עצירות מסננות אמיתית (שרת). **מחלקה** מסומנת `≈` ו"מחיר משוער" (Travelpayouts מחזיר רק תיירות → מכפילים: comfort×1.6, business×3, first×5; הקישורים לאתרי ההזמנה נפתחים במחלקה האמיתית). הכרטיס מציג סה"כ + פירוק "N × לנוסע".
21. **מפת מסלול טיסה** (`focusMap`): סמן מוצא + סמן יעד + **קשת מקווקוות מעוקלת** (`arcPoints` — בזייה ריבועית) + **אייקון מטוס** מסובב לכיוון (`bearing`), `fitBounds` לשתי הנקודות, מתעדכן בשינוי מוצא/יעד. קואורדינטות מ-autocomplete של Travelpayouts (cache לפי IATA).
22. **שמירת מצב ב-F5** (`saveState`/`restoreState`, localStorage `hipusState`): מוצא, יעד, תאריכים, ימי שהייה, מקס׳ עצירות, נוסעים, כבודה, שפה, מטבע, מחלקה, accent, רקע, גודל טקסט.
23. **שורת "החודשים הזולים ביותר"** בתחתית כל חיפוש (`fetchCheapestMonths` → `/flexmonths`, ממוין לפי מחיר, 4 הזולים). **תיקון חשוב:** calendar/flexmonths עברו ל-`one_way=false` (הלוך-חזור) כדי שיתאימו לחיפוש הראשי (קודם הציגו כיוון-אחד → פער מחירים מבלבל).
24. **שמות מותג/כבודה**: "EL AL" באנגלית (לא "El Al"); בעברית כבודה: "תיק טרולי" (יד) / "מזוודה" (רשומה).
